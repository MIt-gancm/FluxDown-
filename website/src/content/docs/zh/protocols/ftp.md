FluxDown 用自带的引擎从 FTP 服务器下载文件,不需要另外的 FTP 客户端。和 HTTP 下载类似,较大的文件会被拆到多个独立连接并行下载,暂停或中断后也会从精确的字节偏移续传。

## 多连接加速

FTP 下载的每个分段都会向服务器打开**自己独立的连接**——这是并行 FTP 的标准做法。FluxDown 把它限制在**最多 4 个并发连接**,不论文件大小或线程设置如何,因为大多数 FTP 服务器只允许每个客户端 IP 5–10 个连接,超过容易被拒绝连接甚至封 IP。1 MB 及以下的文件始终单连接下载,拆分对它们没有帮助。

在启用多连接之前,FluxDown 会先做一次轻量、与文件内容无关的探测,确认服务器是否真的会执行接下来要依赖的 `REST`(断点续传)命令。一些较旧或经过代理的 FTP 服务器会接受 `REST` 命令却仍然从字节 0 开始发送文件——如果对这类服务器使用多连接,下载出来的文件内容会被悄悄打乱。FluxDown 检测到这种情况时,会自动降级为单连接下载。

## 断点续传

FTP 的断点续传依靠 `REST` 命令,告诉服务器从指定的字节偏移开始发送——单连接和多连接下载都靠它实现。下载进度会定期写入 FluxDown 本地数据库,因此暂停任务、关闭应用甚至崩溃,只会丢失几秒钟的进度,而不是整个文件。恢复下载时,单连接下载会从磁盘上已有的部分文件长度处继续;多连接下载则是每个分段各自从自己的断点续传。

## URL 内嵌账号密码

FTP 下载没有单独的用户名/密码输入框——账号密码直接写在 URL 里:

```
ftp://user:pass@host/path/to/file.iso
```

完全省略 `user:pass@` 部分即为匿名登录(FluxDown 默认使用标准的 `anonymous` / `anonymous@` 凭据)。如果密码本身包含 `@`,请把它转义成 `%40`,避免被误判为主机分隔符。

## 代理支持

FTP 下载支持 **SOCKS4、SOCKS5、HTTP 和 HTTPS** 代理,控制连接和数据连接都会经由同一个代理隧道传输。可以在*设置 → 代理*里配置全局代理,也可以在新建下载对话框的高级选项里为单个任务单独指定,格式与 HTTP 下载一致:

```
http://host:port
socks5://host:port
http://user:pass@host:port
socks5://user:pass@host:port
```

单任务代理字段留空时,使用全局代理配置。

## 常见问题与限制

**明明选了更多线程,FTP 下载却只用了一个连接。** 要么文件是 1 MB 及以下,要么 FluxDown 的 REST 探测判断出服务器不能正确支持多段续传——这两种情况都会回退为单连接,以保证文件内容正确而不是一味追求速度。

**为什么 FTP 只限制 4 个连接,HTTP 却能用更多?** FTP 服务器对单 IP 连接数的限制通常比 HTTP/CDN 服务器严格得多;4 是一个保守的上限,用来避免在共享主机上被拒绝连接或封禁。

**匿名 FTP 登录不上。** 检查 URL 里是否残留了空的或格式不对的 `user:pass@` 片段——如果存在但内容有问题,FluxDown 不会自动回退为匿名登录。
